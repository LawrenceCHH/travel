# 全站文章「現代雜誌風格」統一化 — 工作報告（2026-07-21）

## 任務

使用者以資深 UI/UX 設計師角度檢視 `2026-07-20-韓國自由行支付教學.md`（純 Markdown 排版），
認為其呈現效果不如 `2026-07-16-韓國首爾旅行.md`（現代雜誌風格範本）。目標：**在不修改任何
`src/posts/*.md` 內容的前提下**，讓全站文章統一套用現代雜誌風格核心格式，並確認既有 3 篇
文章不會破版。

流程依使用者指定：① 我以資深 UI/UX 設計師身分分析並建立修改計畫 → ② 交給 Opus（資深
UI/UX 設計師視角）審核計畫 → ③ 派資深前端工程師 Agent 依審核後規格改程式碼 → ④ 我本人
獨立驗收成品。

---

## 1. 現況分析

比對三篇現存文章實際使用的呈現機制，發現「風格投入程度」完全不一致：

| 文章 | 機制 | 說明 |
|---|---|---|
| `2026-07-16` 首爾旅行（風格範本） | 手寫 HTML + `.style-a-post` 專屬 class | 整篇包在 `<div class="style-a-post">`，用 `.spot-section`/`.food-item`/`.spot-title` 等專屬詞彙，CSS 定義於 `assets/tailwind.css` L729-946，屬「一次性、opt-in」的客製皮膚 |
| `2026-07-13` 行前準備 | Markdown 卡片 DSL（```food/spot/compare/info/stepper/apps/accordion``` fenced block） | 由 `assets/markdown-cards.js` 的 `registerCardExtensions()` 註冊為 marked.js 擴充，純 Markdown 語法即可產生卡片化排版，已是可重用元件系統 |
| `2026-07-20` 支付教學（使用者不滿意） | 100% 樸素 Markdown（標題/表格/blockquote/checklist） | 未使用上述任何機制，完全落回最基礎的 Tailwind Typography `.prose` 預設樣式 |

**關鍵發現**：所有文章不分格式，一律由 `posts/detail.html` 用 `marked.parse()` 渲染進同一個
`#post-content.prose` 容器（`detail.html` L37、L122-124）。這代表**同一份 `.prose` CSS
規則套用在全站每篇文章**——這是唯一安全、且符合「不動 md 內容」限制的統一槓桿點。

## 2. 決策：排除的方案

- **改寫 07-20 為 DSL fence 或手寫 HTML**：直接違反「不動 md 內容」的硬性限制，排除。
- **JS 啟發式後處理**（自動偵測 `⚠️`/`💡` 開頭段落轉提示框、自動把表格解析重排成卡片）：
  效果可能更強，但 07-20 的表情符號寫在段落內文而非 blockquote、6 張比較表結構互異，
  誤判風險高，且現有 `scripts/verify-card-dsl.mjs` 的 0-diff 驗證機制不涵蓋這種轉換。
  本次排除，列為未來可選 Phase 2。
- **採用方案**：把 07-16 的雜誌感視覺語言（襯線標題階層、pull-quote、克制邊框＋髮絲分隔線、
  暖褐強調色）內化為全站 `.prose` 基礎排版預設值，零內容改動即可讓任何純 Markdown 文章
  自動套用。既有 Card DSL 系統維持不變，作為未來需要更進一步卡片化時的自願升級路徑。

## 3. Opus 審核意見與修正

計畫書先交給 Opus（資深 UI/UX 設計師視角）審核，結論為**有條件核准**，並指出以下需修正之處
（實作時已全數採納）：

1. **最高風險**：新增的 `.prose h3`/`.prose h4` 規則與既有卡片元件規則（`.emergency-card h3`/
   `.app-info h4`，同 specificity）勝負純看程式碼裡誰寫在後面 —— **規則插入位置必須在既有
   `.prose` 區塊內、卡片元件定義之前**，不可附加到檔案末尾，否則會反過來破壞 07-13 的既有卡片
   標題樣式。
2. 07-20 表格若只加大 padding 不縮小字級，會撐寬表格、打回 2026-07-21 稍早才修好的手機橫向
   捲動 bug —— 要求 `td`/`th` 字級降到 14px（小於 body 的 18px）。
3. Blockquote 不應做成有底色的方框（會跟既有 `.alert-note` 提示框視覺撞臉），改為無底色的
   serif pull-quote（左側細線 + 襯線字體）。
4. h4 不應套用 `uppercase`/`tracking-widest`（中文字距拉開很醜），改為中性小標樣式。
5. 07-16 的裸 `<h2>`（總覽/景點漫遊/美食推薦）新樣式若沿用規格原定的 24px，行動版下會比其
   子層 `### Day X`（`.style-a-post h3`，固定 26px）視覺量體更弱，母標題不能弱於子標題。

## 4. 實作內容

**只修改兩個檔案**：`assets/tailwind.css`（+139 行）與 `doc/project.md`（依 CLAUDE.md 規範
補上更新歷史與待辦），**`src/posts/` 下 3 篇文章原始內容一個字都沒有變動**（已用 `git diff`
確認）。

新增規則插入在既有 `.prose td, .prose th { overflow-wrap: break-word; }` 規則之後、
`.toc-sidebar` 定義之前：

- `.prose h1`~`.prose h4` 襯線標題階層：h1 30/34px（行動/桌機）、h2 26/28px、h3 20/21px、
  h4 15px 無襯線小標，h1/h2 皆帶底線分節。
- `.prose blockquote`：拿掉全站 `italic` 設定，改為無底色 serif pull-quote（左側 3px sand 色條）。
- `.prose thead th`/`.prose tbody tr`/`.prose td`：表頭淡底色、髮絲橫線分隔列（無縱線）、
  14px 字級、`tabular-nums` 數字對齊。
- `.prose input[type="checkbox"]`：勾選色改用站內 `--color-primary`。
- `.prose hr`：加大留白至 `3rem 0`。

**實作過程中額外發現並修正的邊界案例**（Opus 審核當下未逐一列出，但工程師驗證時發現）：
07-13 的 `<h3 class="alert-box-title">`（提示框標題）與 07-16「7 大主題景點快速導覽」的
`<h4 class="text-xs uppercase ...">` 都是純 class 選擇器（specificity 0,1,0），天生就低於
新規則的 (0,1,1)，**不論插入順序都會被蓋掉** —— 與靠來源順序牽制的 `.emergency-card h3`
性質不同。修法是另外新增 `.prose h3.alert-box-title` 與 `.prose .editorial-quick-jump h4`
兩條 2-class 選擇器（specificity 0,2,x，穩贏），明確保留這兩處原始的迷你標題外觀。

**唯一偏離 Opus 審核規格的數值**：`.prose h2` 行動版基準由審核建議的 24px 微調為 26px，
以打平 07-16 `.style-a-post h3`（Day X 標題）固定 26px 不隨斷點縮放的量體，避免母子標題
視覺層級倒置。其餘數值均按審核後規格原值實作。
